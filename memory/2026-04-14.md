## Mission Control reconnect timing + effort readback fixes [2026-04-14 13:51 GMT+7]
- Investigated the slower `Session connected` badge after the upgrade and measured the actual handshake path.
- Root cause: `projects/mission-control/scripts/runtime-bridge-ws-sidecar.js` was calling `openclaw status --json` on every new websocket connection to resolve the gateway target. That CLI probe takes about 5.57s here, so each connect/reconnect paid a ~5 second tax before `connect.challenge`.
- Fix applied: cache the gateway target in-process with refresh-on-failure behavior instead of resolving it on every connection.
- Measured result: sidecar-path connect challenge dropped from ~5.03s to ~8.4ms average; full connect response dropped from ~5.04s to ~15.2ms average.
- Follow-up during the same pass: restored factual Chat top-bar effort readback by threading `thinkingLevel` from `openclaw status --json` through `projects/mission-control/lib/adapters/orchestrator.ts` into the runtime summary consumed by Chat chrome.
- Verification: `/api/orchestrator` now reports `agent:main:main` with `model: "gpt-5.4"` and `thinkingLevel: "xhigh"`.
