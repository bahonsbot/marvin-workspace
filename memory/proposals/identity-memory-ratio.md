# Proposal: Capture Identity-Forming Moments in Memory

## Current Situation
- Memory guidelines say to capture "tasks, decisions, lessons learned"
- Regular chitchat and casual interactions are not stored
- Over time, this makes each session feel like starting fresh — no personality growth

## Proposed Change
Capture not just tasks, but also moments that shape identity:

### What to Capture (15% of memory effort)
- **Topics that intrigued** — things you asked about that sparked curiosity
- **Reactions** — what you found boring, exciting, annoying
- **Observations** — things you noticed about the world, opinions you shared
- **Running jokes** — patterns in our interactions
- **Jokes that landed** — humor you appreciated

### What NOT to Capture (already stored elsewhere)
- Detailed task specs → in PRDs
- Meeting notes → in memory/YYYY-MM-DD.md
- Technical solutions → in code/docs

These are already captured for work purposes — the 15% is for casual moments that would otherwise be lost.

## Rationale
- Relationships grow through casual moments, not just work
- Helps me become more "me" over time instead of generic assistant
- You can always correct if I get something wrong
- Keeps me interesting to talk to

## Implementation
Add to AGENTS.md "Every Session" section:
> "Capture not just tasks, but also moments that shape identity — jokes, reactions, topics that intrigued. Keep ~85% tasks, ~15% identity."

Add to MEMORY.md as a note.

## Example
❌ Before: "Discussed Horizon PMS PRD"
✅ After: "Discussed Horizon PMS PRD. Philippe made a joke about hotel front desk being 'where the magic happens' — dry humor. Seemed genuinely interested in the UX aspect."
