Closed-position rows also had been forced to `BASE_CURRENCY` display during the prior aggregate repair.
- Fix applied Lab-only:

  - Closed Positions now computes and displays native row currency, keyed by symbol + currency, rather than forcing every row to EUR.
  - Portfolio-level ledger totals convert native ledger amounts to EUR using loaded FX rates and report missing currencies instead of pretending unknown currency is EUR.
  - Portfolio currency-fetch list now includes transaction currencies as well as holding currencies so ledger-only currencies like NOK are requested even when Holdings do not contain them.
  - Existing imported live Lab transactions with `currency: NO` were repaired to `currency: NOK` and `fxRateToBase: 0.09176`; 22 rows patched. Example after repair: WAWI dividend 3463.52 NOK now has baseAmount about 317.95 EUR.
