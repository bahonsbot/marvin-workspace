## OAuth Refresh Flow Discovery (afternoon session)

### Problem
Session fell back to `bailian/qwen3.5-plus` due to `refresh_token_reused` error on the `openai-codex:default` auth profile. This error occurs when OpenAI rejects a refresh token that has already been used to generate a new access token.

### Key Finding: Two Separate Auth Stores
- **Codex CLI OAuth**: stored in `~/.codex/auth.json` — refreshed via `codex login --device-auth`
- **OpenClaw Runtime OAuth**: stored in `~/.openclaw/agents/main/agent/auth-profiles.json` — refreshed via `openclaw models auth login`

These are independent; refreshing one does not update the other.

### Proper Runtime OAuth Refresh Command
```bash
openclaw models auth login --provider openai-codex --set-default
```

This runs the provider plugin's OAuth flow and applies the default model recommendation. The flow:
1. Generates an OAuth URL for browser authentication
2. User signs in at OpenAI in their local browser
3. User pastes the redirect/callback URL back into the terminal
4. Tokens are saved to `auth-profiles.json`
