# Analysis History

Store one file per substantial analysis or review session to preserve progression over time.

## Naming convention
`YYYY-MM-DD-<market-or-ticker>-analysis-<NN>.md`

Examples:
- `2026-04-10-btcusd-analysis-01.md`
- `2026-04-10-aapl-analysis-02.md`

## Required fields per analysis note
- Date/time
- Market / ticker
- Timeframe(s) reviewed
- Setup or question being assessed
- Risk framing (stop, invalidation, sizing logic, key levels)
- Thesis and conditions
- What changed from prior view
- Next review trigger

## Minimal template
```md
# <date> <market/ticker> analysis <NN>

## Objective

## Context

## Risk framing

## Thesis

## What changed

## Next review trigger
```
